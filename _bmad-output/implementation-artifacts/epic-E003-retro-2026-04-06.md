# Epic E003 Retrospective — Job URL Parsing

**Epic:** `E003`  
**Date:** April 6, 2026  
**Status:** done

## Scope Shipped

- Protected parse-url endpoint in `job-service`
- URL validation and host reachability handling
- Parsed metadata extraction path for job posting sources
- Integration tests for valid and invalid parsing flows

## Wins

- Small, focused epic delivered quickly with low coupling
- Validation-first flow prevented unnecessary network work on malformed URLs
- Error code mapping made parser failures easy to surface through gateway

## Pain Points

- Parsing logic is intentionally lightweight and will need richer extraction heuristics later
- External fetch behavior is a potential latency outlier if left synchronous at scale
- Limited fixture diversity means future source-specific edge cases are still untested

## Decisions That Helped

- Separate parser module from route handler
- Reuse existing auth + error envelope patterns from CRUD work
- Keep alias route support for contract/gateway compatibility

## Action Items

1. Add more parser fixtures from common ATS providers in Wave 2
2. Consider async/offline enrichment for slow or flaky hosts
3. Add observability around parse failure classes before scaling adoption

## Wave 2 Carry-Forward

- Supports richer UX autofill flows
- Candidate for caching and async enrichment in performance roadmap
