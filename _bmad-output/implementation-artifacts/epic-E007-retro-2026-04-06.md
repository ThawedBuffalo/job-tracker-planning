# Epic E007 Retrospective — User Scope Enforcement

**Epic:** `E007`  
**Date:** April 6, 2026  
**Status:** done

## Scope Shipped

- Gateway auth enforcement for protected data routes
- Downstream auth-context propagation with `x-user-id`
- Ownership checks across reads, writes, transitions, and timeline access
- Consistent forbidden envelope mapping and denied-access logging
- Cross-user denial integration coverage

## Wins

- Defense-in-depth approach worked: gateway protects entry, services protect data
- Correlation IDs made denied-access events traceable across layers
- Ownership integration tests captured real risk paths instead of only unit-level checks

## Pain Points

- Some acceptance criteria overlapped with route-map work in `E005`
- Security logging is present but not yet centralized or metrified
- No database-native RLS means policy still lives entirely in app code

## Decisions That Helped

- Keep gateway policy explicit and route-by-route
- Preserve downstream ownership checks instead of trusting gateway alone
- Normalize 401/403 outputs at the gateway boundary
