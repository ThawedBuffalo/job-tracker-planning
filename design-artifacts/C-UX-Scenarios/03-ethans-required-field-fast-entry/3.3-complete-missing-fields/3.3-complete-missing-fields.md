## Overview

**Page Purpose:** Present only the missing required fields in a focused, fast-to-complete form so Ethan can fill the gaps and save without re-reviewing already-filled data.

**User Situation:** Ethan has been shown the missing fields and is now filling just those gaps — minimal typing, keyboard-navigable, no distractions.

**Success Criteria:** Ethan fills only the required missing fields and submits in a few keystrokes.

**Entry Points:**
- "Fill missing fields" CTA from Required Fields Prompt (3.2) — only path

**Exit Points:**
- Submit → Save Application (3.4)
- Back → Required Fields Prompt (3.2)

## Mental State

- **Trigger:** Knows exactly what's missing — just filling the gaps to unblock save
- **Hope:** Two or three inputs, tab through, done — back to batch flow
- **Worry:** Form shows all fields again, not just the missing ones — forces re-review of data already entered

---

## Desired Outcome

**User Goal:** Fill only the missing required fields and submit in under 30 seconds.

**Business Goal:** Ensure the minimum required fields are populated before the record is written to the database.

---

## Variants

None. Fields shown are dynamically determined by what's still missing.
