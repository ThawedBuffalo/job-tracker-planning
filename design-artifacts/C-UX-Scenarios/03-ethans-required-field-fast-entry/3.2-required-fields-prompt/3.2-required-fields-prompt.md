## Overview

**Page Purpose:** Clearly surface exactly which required fields are missing after URL parse, so Ethan knows precisely what he still needs to fill before the record can save.

**User Situation:** Ethan has pasted a URL and the system has extracted what it can — now it's blocking save and showing him exactly what's still missing.

**Success Criteria:** Ethan sees a focused, minimal list of only the missing required fields — no noise, no re-entry of already-filled data.

**Entry Points:**
- URL parse completes from Job URL Paste (3.1) — only path

**Exit Points:**
- Proceed to fill missing fields → Complete Missing Fields (3.3)
- Back → Job URL Paste (3.1)

## Mental State

- **Trigger:** URL parsed — system is asking him to fill gaps before proceeding
- **Hope:** Short list — maybe one or two fields, clearly labelled, easy to tab through
- **Worry:** Long list of required fields that negates the benefit of URL parsing

---

## Desired Outcome

**User Goal:** Know exactly what's missing, nothing more, and get routed directly to filling only those fields.

**Business Goal:** Prevent incomplete records from being saved while keeping friction to a minimum — only block on truly required data.
