## Overview

**Page Purpose:** Provide a fast URL-first entry point where Ethan pastes a job link and the system immediately extracts and prefills as much application data as possible.

**User Situation:** Ethan is batch-adding roles and begins from the add form with a copied job URL ready to paste.

**Success Criteria:** Ethan pastes the URL, sees fields populate instantly, and advances to validation with minimal keystrokes.

**Entry Points:**
- Add application CTA from Dashboard (1.2) — primary path
- FAB / nav shortcut — quick access from anywhere in the app

**Exit Points:**
- Parse completes → Required Fields Prompt (3.2)
- Parse fails → remains on page with manual entry fallback (interaction state)

## Mental State

- **Trigger:** In a focused session adding multiple roles, copying URLs one by one
- **Hope:** Paste URL → system does the heavy lifting → on to the next one
- **Worry:** Has to stop and type out fields manually — breaks the batch rhythm and makes him more likely to skip entries

---

## Desired Outcome

**User Goal:** Paste the URL, see fields populate instantly, and advance to validation with minimal keystrokes.

**Business Goal:** Capture the URL and extract as much structured data as possible, reducing the chance of an abandoned or incomplete record.
