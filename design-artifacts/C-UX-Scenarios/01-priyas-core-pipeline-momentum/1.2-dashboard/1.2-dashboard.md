## Overview

**Page Purpose:** Give Priya an at-a-glance view of her job pipeline and a clear, prominent action to add a new application.

**User Situation:** Priya has just signed in during a focused job-search session and wants to immediately start adding a new role.

**Success Criteria:** Priya sees and activates the add-application action without needing to scan the whole page.

**Entry Points:**
- Successful sign-in from Home (1.1) — primary path
- Direct URL (bookmark) — returning authenticated user

**Exit Points:**
- Add application action → Add Application (1.3)

## Mental State

- **Trigger:** Just signed in, still riding the energy of finding a promising role
- **Hope:** Sees a clear "add job" action immediately — no hunting around
- **Worry:** Dashboard feels cluttered or the add button isn't obvious, breaking her flow

---

## Desired Outcome

**User Goal:** Spot the "add application" action immediately and tap into it without scanning the whole page.

**Business Goal:** Drive the user into the capture flow — every session that results in a new entry is a success.

---

## Variants

None. Two UI states to design for:
- **Empty state** — no applications yet (first use)
- **Populated state** — existing pipeline visible (returning user, Priya's scenario)
