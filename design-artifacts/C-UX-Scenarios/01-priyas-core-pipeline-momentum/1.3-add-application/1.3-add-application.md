## Overview

**Page Purpose:** Provide a fast, low-friction form for capturing a new job application record with the minimum required fields.

**User Situation:** Priya has clicked "add application" from the Dashboard, momentum from finding a promising role still fresh.

**Success Criteria:** Priya fills in the minimum required fields and submits in under a minute.

**Entry Points:**
- Add application button / CTA from Dashboard (1.2) — primary path
- Floating action button (FAB) or nav shortcut — quick access from anywhere in the app

**Exit Points:**
- Successful submit → Confirm Saved (1.4)
- Cancel / back → Dashboard (1.2)

## Mental State

- **Trigger:** Clicked "add" — committed to logging the role right now
- **Hope:** Form is short and smart — minimal typing, sensible defaults, no surprises
- **Worry:** Too many required fields, or unclear what to enter — loses momentum and abandons

---

## Desired Outcome

**User Goal:** Fill in the minimum required fields and submit the record in under a minute.

**Business Goal:** Capture a complete, usable application record that persists to the pipeline.

---

## Variants

None. URL paste treated as an interaction state within this page:
- **Manual entry state** — user types directly into fields (Priya's primary path)
- **URL paste state** — user pastes a job URL, fields auto-populate (interaction detail for specification)
