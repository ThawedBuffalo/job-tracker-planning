## Overview

**Page Purpose:** Present only the remaining required fields that the URL parse couldn't fill, so Ravi can complete the record with minimal effort before saving.

**User Situation:** Ravi has reviewed the parsed data and is now filling only the gaps — typically one or two fields like Status or a field the parser couldn't extract.

**Success Criteria:** Ravi fills the missing required fields in a few keystrokes and submits.

**Entry Points:**
- "Looks good, continue" CTA from Parsed Data Review (4.2) — only path

**Exit Points:**
- Submit → Save Application (4.4)
- Back → Parsed Data Review (4.2)

## Mental State

- **Trigger:** Almost there — filling the last required gap before save
- **Hope:** One or two fields, quick dropdown or short text, then done
- **Worry:** Turns into a long form after already going through review — feels like the parser wasn't worth it

---

## Desired Outcome

**User Goal:** Fill only the missing required field(s) and submit — no re-review of already-confirmed data.

**Business Goal:** Ensure the minimum required data is complete before writing the record, without increasing perceived friction.
