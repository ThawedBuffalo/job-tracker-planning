## Overview

**Page Purpose:** Show Ravi the full result of the URL parse — what was extracted, what mapped to which fields, and what still needs his input — so he can verify accuracy and correct errors before committing the record.

**User Situation:** Ravi has pasted an external URL and the system has finished extraction. He needs a quick, scannable review before proceeding.

**Success Criteria:** Ravi glances at the extracted fields, corrects any errors, and advances to fill any remaining required fields in under 30 seconds.

**Entry Points:**
- Parse completes from Job URL Paste (4.1) — only path

**Exit Points:**
- Proceed to fill required fields → Complete Required Fields (4.3)
- Back → Job URL Paste (4.1) to try a different URL

## Mental State

- **Trigger:** Parse is done — scanning to verify the system got it right
- **Hope:** Fields are accurate and complete — minimal correction needed
- **Worry:** Key fields like job title or company are wrong or blank — loses trust in the tool

---

## Desired Outcome

**User Goal:** Quickly scan extracted fields, make any obvious corrections, and advance to fill only what's missing.

**Business Goal:** Build trust in the parse quality while keeping Ravi moving — correct data at this stage means a clean record downstream.
